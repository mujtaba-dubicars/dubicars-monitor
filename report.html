Findings

Search API hovers at the latency limit

The /api/v2/search endpoint returned 835 ms this run and has ranged ~620–835 ms across runs — consistently near or over the 600 ms bar. The most persistent performance signal so far.

Homepage load rides the 4 s threshold

Homepage load (4111 ms) and the ad page (3923 ms) both sit right at the 4000 ms bar. Once a few days of data exist, tune the bar to a realistic baseline or treat these as an optimization target.

Font blocked by CORS on every page (real, chronic)

assets.dubicars.com/…/tajawal/medium.woff2 returns HTTP 200 but is missing an Access-Control-Allow-Origin header, so browsers reject it. Fix at the CDN by adding the header. Currently suppressed in the monitor so it doesn't alert every run.

Suggestions API is CDN-cached

CloudFront serves the suggestions endpoint from cache and ignores client-side cache-busting, so those timings measure the CDN (~50 ms), not the origin. The monitor flags cached responses in the log's detail column.
